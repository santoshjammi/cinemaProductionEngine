1. Purpose

This Specification defines the canonical Python implementation of the Genesis
Engine runtime, agents, and tooling. It establishes package structure, type
discipline, async model, error handling, and testing requirements so that all
Python contributors produce interoperable code.

2. Python Version and Tooling

- Python 3.11+
- Package manager: `uv` (preferred) or `poetry`
- Lint/format: `ruff` (lint + format)
- Types: `mypy --strict`
- Tests: `pytest` + `pytest-asyncio`
- Build: PEP 621 `pyproject.toml`

3. Package Layout

genesis/
├── pyproject.toml
├── src/genesis/
│   ├── __init__.py
│   ├── runtime/           # orchestrator, workflow engine, API server
│   ├── agents/            # base agent classes + concrete agents
│   ├── ontology/          # OIR + compiler bindings
│   ├── pkg/               # PKG data model, store adapters
│   ├── validation/        # SHACL + JSON Schema + business rules
│   ├── provenance/        # provenance capture and query
│   ├── api/               # REST + GraphQL + SSE
│   ├── cli/               # `genesis` command line
│   ├── config/            # settings, secrets loading
│   └── _internal/         # private helpers
└── tests/

Imports use absolute paths: `from genesis.pkg import Pkg, Node`.

4. Type Discipline

- All public functions and methods MUST have complete type annotations.
- `Any` is forbidden in public APIs. Use generics or `object` where unknown.
- `pydantic v2` models are the canonical runtime representation of PKG nodes
  and edges, generated by the Ontology Compiler.
- `dataclasses` are permitted for value objects that do not require validation.
- Mypy strict mode MUST pass with zero errors.

5. Async Model

- The runtime is async-first: `asyncio` is the only concurrency primitive.
- Sync wrappers are provided for CLI entrypoints only.
- Blocking calls (file I/O, subprocess, model provider HTTP) MUST be wrapped
  in `anyio.to_thread.run_sync` or use async-native clients.
- Cancellation is honored at every `await` point. Long-running agents MUST
  checkpoint progress so cancellation is clean.

6. Agent Base Class

class BaseAgent(ABC):
    agent_id: str
    agent_type: str
    @abstractmethod
    async def invoke(self, ctx: AgentContext, task: TaskAssign) -> TaskResult: ...
    @abstractmethod
    async def validate_inputs(self, task: TaskAssign) -> list[ValidationError]: ...
    async def heartbeat(self) -> None: ...

`AgentContext` carries the session id, PKG handle, provenance recorder, model
client, and logger. Agents MUST NOT access global state outside `ctx`.

7. Error Handling

- Errors follow the GFS-006 error catalog (`specifications/runtime/002`).
- Public functions raise typed exceptions: `GenesisError` subclasses with
  `code`, `message`, `trace_id`, `retriable`.
- Panics (`RuntimeError`, `SystemExit`) MUST be caught at the runtime boundary
  and converted to FATAL AgentMessages.
- Logging uses `structlog` with JSON output. Logs include `trace_id`,
  `session_id`, `production_id`, `agent_id`.

8. Provenance

Every mutation of the PKG MUST be accompanied by a `Provenance` record
capturing agent, session, rationale, evidence ids, confidence, and timestamp.
The provenance recorder is injected via `AgentContext` and is the ONLY legal
way to write provenance.

9. Configuration

- Settings use `pydantic-settings` with environment variable injection.
- Secrets are NEVER loaded from config files; they come from env or a
  platform secrets manager.
- A single `Settings` instance is constructed at runtime start and passed via
  dependency injection. Global settings instances are forbidden.

10. Model Providers

- All model access goes through `genesis.runtime.models.ModelClient`.
- Provider selection is config-driven; agents do not hardcode providers.
- Circuit breakers wrap every provider call. Fallback providers are configured
  per agent type.
- Model calls are retried with exponential backoff up to the agent's deadline.

11. Testing

- Unit tests: pure functions, validators, OIR transforms.
- Integration tests: PKG store, validation pipeline, agent invocations against
  a fixture model client.
- Contract tests: assert emitted artifacts match Ontology Compiler outputs.
- Property tests: `hypothesis` for PKG invariants (edge integrity, confidence
  set).
- Coverage target: 90% line coverage, 80% branch coverage.

12. Performance

- Hot paths (PKG reads, validation) MUST avoid allocations in tight loops.
- `orjson` is the JSON parser.
- Async HTTP uses `httpx` with connection pooling.
- Caching uses `cachetools` or `diskcache` for cross-process needs.

13. Packaging

- The runtime ships as a wheel `genesis-engine`.
- Agents ship as plugins via `genesis.agents` entry points.
- The CLI is the only console script: `genesis = genesis.cli:main`.
