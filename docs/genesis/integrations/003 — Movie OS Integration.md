Authority: Derived from GFS-000

1. Purpose

Defines how Genesis hands off a certified Production Knowledge Package (PKG) to
the Movie OS Studio Engine. Genesis ends at the conclusion of pre-production
(GFS-000 §15); this integration is the single, formal boundary between the two
systems.

2. Scope

- Covers the handoff manifest, the handoff ceremony, and the read-only contract
  the Studio Engine consumes.
- Does not cover what the Studio Engine does after handoff.
- Does not cover media generation; that is the Studio Engine's responsibility,
  not Genesis's.

3. Boundary Principle

- Genesis writes the PKG and the handoff manifest.
- The Studio Engine reads them. It never writes back to the Genesis graph or
  PKG directory.
- If the Studio Engine discovers a defect, it files a defect request and
  Genesis re-enters pre-production for that production. There is no in-place
  mutation across the boundary.

4. Handoff Prerequisites

Handoff is permitted only when all of the following are true (enforced by
GVAL-003 gates G09-G16):

- PKG validation passed (GVAL-001).
- Ontology conformance passed (GVAL-002).
- All quality gates passed (GVAL-003).
- Zero unresolved contradictions.
- Every required decision meets its confidence threshold.
- A Production Readiness Certificate has been issued.
- Zero media artifacts exist in the PKG output.
- A schema-valid handoff manifest has been generated (GEN-HANDOFF).

5. Handoff Manifest

Generated by `genesis generate --kind handoff`. Schema: GSS-NNN (handoff).
The manifest is a JSON document that lists everything the Studio Engine may
consume.

```json
{
  "$schema": "genesis://schemas/json/handoff-manifest.schema.json",
  "handoff_id": "HOFF-<production_id>",
  "production_id": "BRIEF-001",
  "certificate_id": "CERT-<id>",
  "genesis_version": "1.0.0",
  "generated_at": "<ISO 8601>",
  "pkg": {
    "graph_uri": "bolt://localhost:7687/genesis-BRIEF-001",
    "pkg_dir": "pkg/BRIEF-001/",
    "manifest_path": "pkg/BRIEF-001/manifest.yaml"
  },
  "artifacts": [
    {"kind": "production-brief", "path": "pkg/BRIEF-001/brief.yaml", "schema": "GSS-NNN"},
    {"kind": "character-dna", "id": "CHAR-001", "path": "pkg/BRIEF-001/characters/ethan-morrison.json", "schema": "GSS-NNN"},
    {"kind": "character-dna", "id": "CHAR-002", "path": "pkg/BRIEF-001/characters/lena-morrison.json", "schema": "GSS-NNN"},
    {"kind": "scene-specification", "id": "SCENE-001", "path": "pkg/BRIEF-001/scenes/scene-001.json", "schema": "GSS-NNN"},
    {"kind": "world-bible", "path": "pkg/BRIEF-001/world/world-bible.json", "schema": "GSS-NNN"},
    {"kind": "narrative-outline", "path": "pkg/BRIEF-001/narrative/outline.json", "schema": "GSS-NNN"}
  ],
  "constraints": {
    "target_duration_minutes": 12,

    "visual_style": "cinematic, shallow depth of field",
    "narration_voice": "en-US-GuyNeural",
    "music_style": "ambient piano, minimal"
  },
  "read_only": true,
  "license": "Genesis-certified pre-production knowledge. Studio Engine may read; may not modify."
}
```

6. Handoff Ceremony

1. Genesis certifies the PKG (`genesis certify`).
2. Genesis generates the handoff manifest (`genesis generate --kind handoff`).
3. Genesis writes the manifest to `pkg/<production_id>/handoff.json`.
4. Genesis emits a `HandoffReady` event with the manifest path.
5. The Studio Engine picks up the event, reads the manifest, and begins its
   own pipeline.
6. Genesis records the handoff in the PKG manifest and marks the production as
   `HandedOff`. No further Genesis writes occur.

7. Studio Engine Contract

The Studio Engine may:
- Read every artifact listed in the manifest.
- Read the Genesis graph via the declared `graph_uri` (read-only credentials).
- Use the constraints block to configure its pipeline.

The Studio Engine may not:
- Write to the Genesis graph or PKG directory.
- Modify any artifact listed in the manifest.
- Re-interpret the synopsis; if intent is missing, it must request a Genesis
  re-run.

8. Defect Loop

If the Studio Engine finds a defect (e.g., a scene spec missing a beat):
1. Studio Engine emits a `DefectReport` with the artifact id and the gap.
2. Genesis opens a new pre-production task for that production.
3. Genesis re-runs the relevant workflow stages under the same production ID.
4. Genesis re-certifies and re-issues the handoff manifest with a new
   `handoff_id` and a revised `genesis_version`.

9. Versioning

- The handoff manifest is versioned by `generated_at` and `genesis_version`.
- The Studio Engine records which manifest version it consumed for each
  downstream artifact it produces.
- This enables traceability from a final rendered frame back to the exact PKG
  version that authorized it.

10. Dependencies

- Validators: GVAL-001, GVAL-002, GVAL-003
- Generator: GGEN-001 (GEN-HANDOFF)
- CLI: GTOOL-001 (`genesis certify`, `genesis generate --kind handoff`)
- Graph: GINT-001 (read-only access for Studio Engine)
- Tests: GTEST-005 (T016 handoff manifest present)
