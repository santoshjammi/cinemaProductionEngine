1. Purpose

This document summarizes the Genesis Ontology family. It lists every ontology with its domain, purpose, and key concepts. It is the index for the vocabulary layer.

For full text, see `ontology/<domain>/`.

2. Ontology Families

The GO family is organized into four tiers:

- Core (GO-001..006) — universal vocabulary.
- Domain (GO-101..119) — domain-specific extensions.
- Specialized (GO-201+) — grammar-specific extensions.
- Production (GO-301+) — production-specific extensions.
- Generated (GO-200+) — compiler-derived ontologies.

7. Generated Ontologies (GO-200+)

Generated ontologies are derived by the Genesis Compiler from non-ontology sources (schemas, specifications, manifests, pattern libraries). They are registered in `ontology/generated/001 — Generated Ontology Registry.md`. They do not hold constitutional authority.

8. Inheritance Rules

- A domain concept may extend a Core concept via `derives_from`.
- A domain concept may add properties but may not remove or rename Core properties.
- A specialized concept may extend a domain concept following the same rules.
- A production concept may extend a domain or specialized concept.
- Generated concepts conform to their parent ontology but do not modify it.

9. Vocabulary Discipline

- Concept names: PascalCase, singular, English.
- Property names: camelCase.
- Relationship names: snake_case.
- Enumerations: UPPER_SNAKE_CASE.
- IDs: GO-NNN, zero-padded within tier.

10. Lifecycle

All ontology concepts follow GO-003: Draft → Proposed → Validated → Approved → Certified → Superseded. Ontology-level changes require governance approval.

11. Conflict Resolution

When a domain ontology appears to conflict with the Core Ontology, the Core Ontology prevails. Conflicts are recorded as validation findings and routed to the Governance Engine.

12. Extension Policy

- New Core concepts require a constitutional amendment.
- New domain concepts require a domain ontology specification and governance approval.
- New specialized concepts require a specialized ontology specification and a derivation chain.
- New production concepts require a production ontology specification and a derivation chain.
- New generated ontologies require a compiler run and a registry entry.

13. Approval

This overview is binding as an index. For any conflict, the full text of the relevant GO-NNN ontology prevails.
